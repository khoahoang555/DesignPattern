2. 💣 Ví Dụ Code Khó Khăn (Trước khi áp dụng pattern)

// ❌ Code "bẩn" — orderPizza() biết quá nhiều thứ  
public Pizza orderPizza(String type) {  
    Pizza pizza;  

    // Mỗi lần thêm/xóa pizza phải sửa chỗ này  
    if (type.equals("cheese")) {  
        pizza = new CheesePizza();  
    } else if (type.equals("greek")) {  
        pizza = new GreekPizza();       // Hôm nay phải xóa cái này  
    } else if (type.equals("pepperoni")) {  
        pizza = new PepperoniPizza();  
    } else if (type.equals("clam")) {  
        pizza = new ClamPizza();        // Hôm nay phải thêm cái này  
    } else if (type.equals("veggie")) {  
        pizza = new VeggiePizza();      // Hôm nay phải thêm cái này  
    }  

    // Phần này không thay đổi, nhưng bị "nhiễm" bởi phần trên  
    pizza.prepare();  
    pizza.bake();  
    pizza.cut();  
    pizza.box();  
    return pizza;  
}

⚠️ Vấn đề: Phần if-else (khởi tạo) thay đổi liên tục, còn phần prepare/bake/cut/box thì không — nhưng chúng đang bị nhốt chung trong một method.

3. 🛠️ Cách Pattern Giải Quyết Vấn Đề

Factory Pattern giải quyết bằng cách tách biệt phần khởi tạo object ra khỏi phần sử dụng object, theo nguyên tắc "Identify the aspects that vary and separate them from what stays the same."

Có 3 cấp độ được giới thiệu trong chương:

Simple Factory  →  Factory Method Pattern  →  Abstract Factory Pattern  
  (idiom)              (pattern)                    (pattern)

🔹 Simple Factory

Đơn giản nhất: tạo một class chuyên trách việc khởi tạo pizza.

🔹 Factory Method Pattern

Dùng abstract method trong superclass, để subclass quyết định sẽ tạo loại pizza nào. orderPizza() không cần biết pizza cụ thể là gì.

🔹 Abstract Factory Pattern

Tạo ra một interface để khởi tạo cả một "gia đình" sản phẩm (dough, sauce, cheese...). Mỗi region có factory riêng, đảm bảo nguyên liệu đồng nhất và chất lượng.

4. ✅ Ví Dụ Code Đã Áp Dụng Pattern

🔹 Simple Factory

// Factory chuyên tạo pizza  
public class SimplePizzaFactory {  
    public Pizza createPizza(String type) {  
        Pizza pizza = null;  
        if (type.equals("cheese"))   pizza = new CheesePizza();  
        else if (type.equals("clam"))    pizza = new ClamPizza();  
        else if (type.equals("veggie"))  pizza = new VeggiePizza();  
        return pizza;  
    }  
}  

// PizzaStore chỉ cần dùng factory, không cần biết pizza cụ thể  
public class PizzaStore {  
    SimplePizzaFactory factory;  

    public PizzaStore(SimplePizzaFactory factory) {  
        this.factory = factory;  
    }  

    public Pizza orderPizza(String type) {  
        Pizza pizza = factory.createPizza(type); // ✅ Không còn new trực tiếp  
        pizza.prepare();  
        pizza.bake();  
        pizza.cut();  
        pizza.box();  
        return pizza;  
    }  
}

🔹 Factory Method Pattern

// Abstract PizzaStore — định nghĩa "khung" quy trình  
public abstract class PizzaStore {  
    public Pizza orderPizza(String type) {  
        Pizza pizza = createPizza(type); // Gọi factory method  
        pizza.prepare();  
        pizza.bake();  
        pizza.cut();  
        pizza.box();  
        return pizza;  
    }  

    // ✅ Subclass sẽ quyết định tạo pizza nào  
    protected abstract Pizza createPizza(String type);  
}  

// NY Store tự quyết định pizza của mình  
public class NYPizzaStore extends PizzaStore {  
    @Override  
    protected Pizza createPizza(String type) {  
        if (type.equals("cheese"))  return new NYStyleCheesePizza();  
        else if (type.equals("clam"))   return new NYStyleClamPizza();  
        else return null;  
    }  
}  

// Chicago Store tự quyết định pizza của mình  
public class ChicagoPizzaStore extends PizzaStore {

        else if (type.equals("clam"))   return new ChicagoStyleClamPizza();

// Pizza dùng factory để lấy nguyên liệu — không quan tâm region  
public class CheesePizza extends Pizza {

    @Override  
    void prepare() {  
        dough  = ingredientFactory.createDough();   // ✅ Lấy từ factory

}

5. 📖 Định Nghĩa Của Các Pattern

Factory Method Pattern: "Defines an interface for creating an object, but lets subclasses decide which class to instantiate. Factory Method lets a class defer instantiation to subclasses."

Abstract Factory Pattern: "Provides an interface for creating families of related or dependent objects without specifying their concrete classes."

Simple Factory: Không phải Design Pattern chính thức, mà là một programming idiom — một kỹ thuật lập trình phổ biến nhưng không có cấu trúc pattern đầy đủ.

7. ⚠️ Những Điều Cần Chú Ý

1. Dependency Inversion Principle

"High-level components should not depend on low-level components; both should depend on abstractions."

Cần nội hóa 3 nguyên tắc sau (dù trong thực tế không thể tuân thủ 100%):

  • Không nên giữ reference đến concrete class (tránh dùng new trực tiếp)
  • Không nên kế thừa từ concrete class
  • Không nên override một implemented method của base class

2. Simple Factory ≠ Factory Method

Hai cái trông giống nhau nhưng khác về bản chất:

  • Simple Factory: object tạo pizza được compose vào PizzaStore
  • Factory Method: subclass kế thừa và override method tạo pizza

3. Abstract Factory thường dùng Factory Method bên trong

"Often the methods of an Abstract Factory are implemented as factory methods." — Đây là điều hoàn toàn tự nhiên.

4. Parameterized Factory cần xử lý type-safety

Khi dùng String để phân loại pizza, lỗi chỉ phát hiện lúc runtime. Nên cân nhắc dùng enum hoặc static constants để bắt lỗi sớm hơn ở compile time.

5. Không nhất thiết phải có nhiều ConcreteCreator

Factory Method vẫn có giá trị ngay cả khi chỉ có 1 concrete creator, vì nó decouples implementation khỏi usage — thay đổi product sẽ không ảnh hưởng Creator.
