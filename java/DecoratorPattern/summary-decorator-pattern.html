Chương 3: Decorator Pattern — Tổng Hợp Chi Tiết

1. 🧩 Khó Khăn Khi Không Áp Dụng Pattern Này

Câu chuyện: Starbuzz Coffee

Starbuzz Coffee là một chuỗi cà phê phát triển cực nhanh. Ban đầu, họ thiết kế hệ thống đặt hàng với một lớp Beverage (đồ uống) cơ sở, và các lớp con như Espresso, DarkRoast, HouseBlend, v.v.

Vấn đề nảy sinh khi khách hàng muốn thêm các condiment (topping) như sữa hấp (steamed milk), soy, mocha, whipped cream... và mỗi condiment có giá riêng.

Cách tiếp cận đầu tiên (dùng inheritance thuần túy):

  • Tạo ra các lớp con như: EspressoWithMocha, EspressoWithMochaAndWhip, DarkRoastWithSoy, DarkRoastWithSoyAndMocha...
  • Kết quả: bùng nổ số lượng class (class explosion) — hàng chục, thậm chí hàng trăm lớp con chỉ để biểu diễn mọi tổ hợp có thể.

Cách tiếp cận thứ hai (dùng boolean flags trong base class):

  • Thêm các biến hasMilk, hasSoy, hasMocha, hasWhip vào Beverage cùng các getter/setter.
  • Tính giá trong cost() bằng cách kiểm tra từng flag.

2. 💻 Ví Dụ Code Khi Chưa Áp Dụng Pattern

// Base class với boolean flags — cách tiếp cận tệ  
public abstract class Beverage {  
    String description;  
    boolean milk;  
    boolean soy;  
    boolean mocha;  
    boolean whip;  

    public double cost() {  
        double condimentCost = 0.0;  
        if (hasMilk())  condimentCost += 0.10;  
        if (hasSoy())   condimentCost += 0.15;  
        if (hasMocha()) condimentCost += 0.20;  
        if (hasWhip())  condimentCost += 0.10;  
        return condimentCost;  
    }  

    public boolean hasMilk() { return milk; }  
    public void setMilk(boolean milk) { this.milk = milk; }  
    // ... tương tự cho soy, mocha, whip  
}  

// Lớp con  
public class DarkRoast extends Beverage {  
    public DarkRoast() {  
        description = "Dark Roast Coffee";  
    }  

    @Override  
    public double cost() {  
        return 0.99 + super.cost(); // Chỉ cộng thêm giá base  
    }  
}  

// Vấn đề: Nếu thêm "Oat Milk" mới → phải sửa Beverage + mọi lớp con!

Vấn đề rõ ràng:

  • Thêm topping mới → sửa Beverage (vi phạm Open-Closed Principle)
  • Không thể đặt double mocha (2 lần mocha) vì chỉ có 1 boolean flag

3. 🔧 Cách Decorator Pattern Giải Quyết Vấn Đề

Thay vì kế thừa để thêm hành vi, ta bọc (wrap) object bằng các decorator object tại runtime.

Nguyên tắc cốt lõi:

Ưu tiên object composition hơn class inheritance để thêm hành vi mới.

Quy trình xây dựng đơn hàng với Decorator:

Khách muốn: DarkRoast + Mocha + Whip  

Bước 1: Tạo DarkRoast object  
Bước 2: Bọc nó trong Mocha decorator  
Bước 3: Bọc Mocha trong Whip decorator  
Bước 4: Gọi cost() trên outermost decorator (Whip)  
         → Whip.cost() gọi Mocha.cost()  
         → Mocha.cost() gọi DarkRoast.cost()  
         → Tổng hợp trả về

Những điều Decorator đảm bảo:

  • Decorator có cùng supertype với object nó bọc → có thể dùng thay thế nhau
  • Có thể dùng một hoặc nhiều decorator để bọc một object
  • Hành vi được thêm before/after khi delegate xuống object bên trong
  • Có thể decorate bất kỳ lúc nào tại runtime

4. 💻 Ví Dụ Code Đã Áp Dụng Decorator Pattern

// ===== ABSTRACT COMPONENT =====  
public abstract class Beverage {  
    String description = "Unknown Beverage";  

    public String getDescription() {  
        return description;  
    }  

    public abstract double cost();  
}  

// ===== ABSTRACT DECORATOR =====  
public abstract class CondimentDecorator extends Beverage {  
    Beverage beverage; // giữ reference đến object được wrap  

    public abstract String getDescription();  
}  

// ===== CONCRETE COMPONENTS =====  
public class Espresso extends Beverage {  
    public Espresso() {  
        description = "Espresso";  
    }  

    @Override  
    public double cost() { return 1.99; }  
}  

public class DarkRoast extends Beverage {  
    public DarkRoast() {  
        description = "Dark Roast Coffee";  
    }  

    @Override  
    public double cost() { return 0.99; }  
}  

public class HouseBlend extends Beverage {  
    public HouseBlend() {  
        description = "House Blend Coffee";  
    }  

    @Override  
    public double cost() { return 0.89; }  
}  

// ===== CONCRETE DECORATORS =====  
public class Mocha extends CondimentDecorator {  
    public Mocha(Beverage beverage) {  
        this.beverage = beverage;  
    }  

    @Override  
    public String getDescription() {  
        return beverage.getDescription() + ", Mocha";  
    }  

    @Override  
    public double cost() {  
        return beverage.cost() + 0.20; // thêm giá Mocha  
    }  
}  

public class Whip extends CondimentDecorator {  
    public Whip(Beverage beverage) {  
        this.beverage = beverage;  
    }  

    @Override  
    public String getDescription() {  
        return beverage.getDescription() + ", Whip";  
    }  

    @Override  
    public double cost() {  
        return beverage.cost() + 0.10;  
    }  
}  

public class Soy extends CondimentDecorator {  
    public Soy(Beverage beverage) {  
        this.beverage = beverage;  
    }  

    @Override  
    public String getDescription() {  
        return beverage.getDescription() + ", Soy";  
    }  

    @Override  
    public double cost() {  
        return beverage.cost() + 0.15;  
    }  
}  

// ===== SỬ DỤNG =====  
public class StarbuzzCoffee {  
    public static void main(String[] args) {  

        // Espresso thuần, không topping  
        Beverage b1 = new Espresso();  
        System.out.println(b1.getDescription() + " $" + b1.cost());  
        // → Espresso $1.99  

        // DarkRoast + Double Mocha + Whip  
        Beverage b2 = new DarkRoast();  
        b2 = new Mocha(b2);  
        b2 = new Mocha(b2); // Double Mocha!  
        b2 = new Whip(b2);  
        System.out.println(b2.getDescription() + " $" + b2.cost());  
        // → Dark Roast Coffee, Mocha, Mocha, Whip $1.49  

        // HouseBlend + Soy + Mocha + Whip  
        Beverage b3 = new HouseBlend();  
        b3 = new Soy(b3);  
        b3 = new Mocha(b3);  
        b3 = new Whip(b3);  
        System.out.println(b3.getDescription() + " $" + b3.cost());  
        // → House Blend Coffee, Soy, Mocha, Whip $1.34  
    }  
}

💡 Double Mocha hoàn toàn khả thi chỉ bằng cách wrap 2 lần — điều không thể làm với boolean flag.

5. 📖 Định Nghĩa Của Decorator Pattern

"The Decorator Pattern attaches additional responsibilities to an object dynamically. Decorators provide a flexible alternative to subclassing for extending functionality."

Các thành phần trong pattern:

Thành phầnVai tròVí dụ trong Starbuzz
Abstract ComponentInterface/abstract class chungBeverage
Concrete ComponentObject gốc được decorateEspresso, DarkRoast
Abstract DecoratorGiữ reference đến Component, cùng typeCondimentDecorator
Concrete DecoratorThêm hành vi cụ thểMocha, Whip, Soy

Nguyên tắc thiết kế được áp dụng:

  • Open-Closed Principle: Mở để mở rộng, đóng để sửa đổi
  • Favor Composition over Inheritance: Hành vi đến từ việc kết hợp object, không phải kế thừa

7. ⚠️ Những Điểm Cần Chú Ý Khi Sử Dụng

🔴 Không nên kiểm tra concrete type

// ❌ SAI — sẽ bị phá vỡ khi wrap bằng decorator  
if (beverage instanceof HouseBlend) {  
    applyDiscount(beverage); // sẽ KHÔNG chạy nếu HouseBlend đã được wrap!  
}

Chỉ nên viết code dựa trên abstract component type. Nếu phụ thuộc vào concrete type, cần xem lại thiết kế.

🔴 Không áp dụng Open-Closed Principle ở mọi nơi

Áp dụng nguyên tắc này khắp nơi là lãng phí và không cần thiết, có thể dẫn đến code phức tạp, khó hiểu. Chỉ áp dụng ở những điểm dễ thay đổi nhất.

🔴 Kết hợp với Factory/Builder Pattern khi tạo decorated objects

Nên dùng Factory hoặc Builder Pattern để tạo các decorated object — tránh việc tạo thủ công dễ gây nhầm lẫn về thứ tự wrap.

🔴 Decorator không nên "nhìn xuyên" chuỗi

Khi cần biết tất cả decorator trong chuỗi (VD: in "Whip, Double Mocha" thay vì "Mocha, Whip, Mocha"), bạn đang vượt quá mục đích thực sự của Decorator. Cần cân nhắc dùng giải pháp khác như getDescription() trả về ArrayList.

🔡 Thực tế: Java I/O dùng Decorator

Java đã áp dụng Decorator Pattern trong toàn bộ package java.io:

// Ví dụ: Đọc file với BufferedInputStream (decorator) bọc FileInputStream  
InputStream in = new LowerCaseInputStream(  
                     new BufferedInputStream(  
                         new FileInputStream("test.txt")));

InputStream = Abstract Component; FileInputStream = Concrete Component; BufferedInputStream, LowerCaseInputStream = Concrete Decorators.
